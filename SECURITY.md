# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Ral, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, use [GitHub Security Advisories](https://github.com/ral-workspace/ral/security/advisories/new) to report privately.

Include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Supported Versions

| Version | Supported |
| --- | --- |
| Latest release | Yes |
| Older versions | No |

## Scope

This policy covers the Ral desktop application and its Rust/TypeScript source code in this repository. Third-party dependencies are managed via `pnpm` and `cargo` and are subject to their own security policies.
